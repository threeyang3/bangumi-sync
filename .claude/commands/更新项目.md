# 更新项目

当用户发送"更新项目"时，依次执行以下步骤：

## 版本号规则

- **小版本更新**（默认）：版本号增加 0.1（如 4.1.0 → 4.2.0）
- **大版本更新**（需特别声明"更新大版本"）：大版本号 +1，小版本号归零（如 4.1.0 → 5.0.0）

## 步骤 1: 更新版本号

1. 读取 `manifest.json` 获取当前版本号
2. 计算新版本号
3. 同时更新以下文件中的版本号：
   - `manifest.json`
   - `package.json`

## 步骤 2: 更新文档

### 更新 README.md

README.md 结构要求：
1. **主要功能** - 描述所有已实现的功能
2. **最新更新** - 只描述最新一次更新的内容

### 更新 CLAUDE.md

1. 确认版本历史部分已更新
2. 确认功能描述与当前实现一致
3. 确认项目结构文档反映最新文件

## 步骤 3: 构建并发布

### 3.1 编译项目

```bash
npm run build
```

### 3.2 创建发布包

1. 在 `release/` 下创建版本号命名的子目录
2. 复制文件到该目录
3. 创建 zip 压缩包到 `archives/` 目录

```bash
# 创建版本目录
mkdir -p release/v{版本号}

# 复制文件
cp main.js release/
cp manifest.json release/
cp styles.css release/
cp main.js release/v{版本号}/
cp manifest.json release/v{版本号}/
cp styles.css release/v{版本号}/

# 创建 zip 到 archives
cd archives && powershell -Command "Compress-Archive -Path ../release/main.js,../release/manifest.json,../release/styles.css -DestinationPath bangumi-sync-v{版本号}.zip -Force"
```

### 3.3 提交到 Git

```bash
git add -A
git commit -m "release: v{版本号} 版本说明"
git push
```

### 3.4 创建 GitHub Release

**重要**: Release tag 必须与 manifest.json 版本号完全一致，不带 `v` 前缀。

```bash
gh release create {版本号} ./release/main.js ./release/manifest.json ./release/styles.css --title "v{版本号}" --notes "更新内容说明"
```

**Release 资产要求**：
- 必须包含单独的文件：`main.js`、`manifest.json`、`styles.css`
- 不能只上传 zip 文件
- Tag 必须是纯版本号（如 `4.3.0`），不能带 `v` 前缀

### 3.5 更新 Obsidian 社区插件仓库

如果这是首次发布或需要更新插件信息，执行以下步骤：

#### 3.5.1 Fork 并克隆仓库

```bash
# Fork 仓库（首次）
gh repo fork obsidianmd/obsidian-releases --clone=false

# 克隆 fork 到本地
git clone --depth 1 https://github.com/threeyang3/obsidian-releases.git /tmp/obs-rel
```

#### 3.5.2 更新 community-plugins.json

```bash
cd /tmp/obs-rel

# 使用 Python 更新 JSON
python3 -c "
import json

with open('community-plugins.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

# 查找并更新插件信息
for plugin in data:
    if plugin['id'] == 'bangumi-sync':
        plugin['description'] = '新描述内容'  # 如果描述有变化
        break

with open('community-plugins.json', 'w', encoding='utf-8') as f:
    json.dump(data, f, indent=2, ensure_ascii=False)
"
```

#### 3.5.3 提交并创建 PR

```bash
cd /tmp/obs-rel
git config user.email "threeyang3@users.noreply.github.com"
git config user.name "threeyang"
git add community-plugins.json
git commit -m "Update Bangumi Sync plugin"
git push origin master

# 创建 PR
gh pr create --repo obsidianmd/obsidian-releases \
  --title "Update Bangumi Sync plugin" \
  --body "$(cat <<'EOF'
# I am submitting a new Community Plugin

- [x] I attest that I have done my best to deliver a high-quality plugin...

## Repo URL
Link to my plugin: https://github.com/threeyang3/bangumi-sync

## Release Checklist
- [x] I have tested the plugin on Windows
- [x] My GitHub release contains all required files
- [x] GitHub release name matches the exact version number in manifest.json
- [x] The `id` in my `manifest.json` matches the `id` in `community-plugins.json`
- [x] My README.md describes the plugin's purpose and usage
- [x] I have added a license in the LICENSE file
EOF
)"
```

## 完整流程示例

```bash
# 1. 构建项目
npm run build

# 2. 创建发布目录和复制文件
mkdir -p release/v4.2.0
cp main.js manifest.json styles.css release/
cp main.js manifest.json styles.css release/v4.2.0/

# 3. 创建压缩包（可选，用于备份）
cd archives && powershell -Command "Compress-Archive -Path ../release/main.js,../release/manifest.json,../release/styles.css -DestinationPath bangumi-sync-v4.2.0.zip -Force"

# 4. 提交并推送
git add -A
git commit -m "release: v4.2.0 功能描述"
git push

# 5. 创建 GitHub Release（注意：tag 不带 v 前缀）
gh release create 4.2.0 ./release/main.js ./release/manifest.json ./release/styles.css --title "v4.2.0" --notes "更新内容"
```

## 注意事项

- 确保所有代码修改已完成并编译通过
- 确保版本号在 `manifest.json` 和 `package.json` 中都已更新
- `release/` 目录下的三个文件应为最新编译结果
- GitHub Release 需要使用 `gh` CLI 工具
- 历史版本压缩包存放在 `archives/` 目录
