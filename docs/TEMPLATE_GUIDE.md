# 模板设计参考

相关文档：

- 模块与数据流：见 [ARCHITECTURE.md](./ARCHITECTURE.md)
- 字段分类和同步分支判断：见 [LOGIC_REFERENCE.md](./LOGIC_REFERENCE.md)

本文档说明 Bangumi Sync 当前版本实际支持的模板能力、变量、默认值规则与设计建议。内容以当前代码行为为准，适用于：

- 内置标准模板
- 内置作者模板
- 从文件选择的自定义模板
- 设置面板中直接编辑的自定义模板内容

## 1. 模板的作用范围

模板决定的是“本地笔记长什么样”。

插件会在同步时把 Bangumi 条目数据、收藏数据、章节显示、相关条目链接，以及用户在弹窗里补充的本地属性值渲染进模板，然后写成 Markdown 文件。

模板主要影响：

- frontmatter 字段结构
- 正文排版
- 哪些本地自定义属性需要用户补充
- 这些自定义属性的默认值是什么

## 2. 模板来源与选择逻辑

每种条目类型都可以单独选择模板来源：

- 标准模板
- 作者自用模板
- 从文件选择
- 自定义内容

书籍类型会进一步按解析出的细分类别选择模板：

- 小说：使用 `novel` 模板
- 漫画：使用 `comic` 模板
- 画集 / 画本：优先使用 `album` 模板

如果没有命中特化模板，才会回退到该类型的默认模板。

## 3. 模板基础语法

### 3.1 变量替换

使用 `{{变量名}}`：

```markdown
---
中文名: "{{name_cn}}"
Bangumi评分: "{{rating}}"
---
```

如果变量为空，会渲染为空字符串。

### 3.2 条件渲染

使用 `{{#if 变量}}...{{/if}}`：

```markdown
{{#if related}}相关:
{{related}}
{{/if}}
```

只有当变量非空时，这一整段才会输出。

### 3.3 默认值语法

使用 `{{变量|默认值}}`：

```markdown
评分: {{my_rate|0}}
```

当变量为空时，输出你写的默认值。

## 4. 模板中的“默认值”应该怎么写

自定义属性默认值不再来自插件设置面板，而是直接来自模板 frontmatter 中写死的字面量。

例如：

```markdown
---
游玩状态: {{my_status}}
tags:
{{tags}}
评分: {{my_rate}}
平台: "Steam"
渠道: "实体"
已购: false
资源属性: []
标语:
存储:
---
```

这里的行为是：

- `平台` 默认值是 `Steam`
- `渠道` 默认值是 `实体`
- `已购` 默认值是布尔 `false`
- `资源属性` 默认值是空列表
- `标语` 默认是空
- `存储` 默认是空

当同步前弹出“本地自定义属性”弹窗时：

- 这些字面量会作为默认值显示
- 用户点击“跳过”时，会直接保留这些模板默认值
- 用户手动修改后，会用用户输入覆盖模板默认值

## 5. 动态属性弹窗如何识别字段

插件会从模板 frontmatter 中动态识别“需要用户补充的本地属性”。当前逻辑不是手写一份固定字段名单，而是：

1. 先取出模板 frontmatter 里的全部属性
2. 再过滤掉插件能自动写入的辨识属性和 Bangumi / 同步流程可提供的属性
3. 剩下的字段都视为本地自定义属性

具体规则如下：

- 必须位于 frontmatter 中
- 可以是空字段，例如 `存储:`
- 可以是字面量字段，例如 `平台: "Steam"`、`已购: false`
- 会过滤掉辨识属性，例如 `id`、`中文名`、`原名`、`作品大类`、`具体类型`
- 会过滤掉自动字段，例如 `Bangumi链接`、`封面`
- 如果属性值只依赖 Bangumi / 同步流程可自动提供的变量，也会过滤掉
  例如 `评分: {{my_rate}}`、`作者: "{{author}}"`、`平台: "{{platform}}"`
- 如果属性值不是自动变量，或者是空字段 / 字面量字段，就会进入弹窗
  例如 `剧情评分: {{rating_story}}`、`平台: "Steam"`、`存储:`

字段类型判定：

- `true` / `false` 会渲染成开关
- `[]` 会渲染成列表输入框，输入时用英文逗号 `,` 分隔多个值
- 其他字面量或空值会渲染成文本输入框

不会进入动态属性弹窗的内容：

- 收藏状态
- 标签
- 短评
- 正文中的 `记录`
- 正文中的 `感想`

## 6. 当前支持的模板变量

下面列的是当前代码里真实提供的变量。

### 6.1 基础信息

| 变量 | 说明 |
|------|------|
| `{{id}}` | 条目 ID |
| `{{name}}` | 原名 |
| `{{name_cn}}` | 中文名 |
| `{{alias}}` | 从 infobox 提取的别名（支持字符串和数组格式，多个用顿号分隔） |
| `{{summary}}` | 简介，已做基础清洗 |
| `{{rating}}` | Bangumi 平均分 |
| `{{rank}}` | Bangumi 排名 |
| `{{cover}}` | 封面地址，本地或网络 |
| `{{bangumi_url}}` | 条目链接 |

### 6.2 类型与日期

| 变量 | 说明 |
|------|------|
| `{{type}}` | 条目类型大类（小写，如 book/anime/music/game/real） |

| `{{year}}` | 年 |
| `{{month}}` | 月 |

| 大类 | category 值 |

## 7. 评分明细变量

这些变量现在不再被插件单独当作“评分明细类型”处理，而是和其他本地自定义属性一样，由模板属性解析结果统一决定是否需要用户填写。

推荐写法：

```markdown
剧情评分:
趣味评分:
```

这样它们会和 `标语`、`资源属性`、`版本` 一样，作为普通自定义属性出现在弹窗里。

旧模板里的这种写法仍然兼容：

```markdown
剧情评分: {{rating_story}}
```

但这已经只是兼容旧模板的输入变量写法，不再是内置模板的默认推荐写法。

通用变量名如下：

| 变量 | 说明 |
|------|------|
| `{{rating_music}}` | 音乐评分 |
| `{{rating_character}}` | 人设评分 |
| `{{rating_story}}` | 剧情评分 |
| `{{rating_art}}` | 美术评分 |
| `{{rating_illustration}}` | 插画评分 |
| `{{rating_writing}}` | 文笔评分 |
| `{{rating_drawing}}` | 画工评分 |
| `{{rating_fun}}` | 趣味评分 |

是否填写、填写哪些项，不再由某个窗口里写死的类型判断决定，而是由模板 frontmatter 里实际出现了哪些属性决定。

## 8. 章节显示变量

| 变量 | 说明 |
|------|------|
| `{{episodes}}` | 动画 / 漫画的章节显示 HTML |
| `{{volumes_display}}` | 小说卷显示 HTML |

这些变量会输出一组 `.ep-box` HTML 片段，适合直接放进正文。

例如：

```markdown
## 进度

{{episodes}}
```

## 9. 相关条目变量

| 变量 | 说明 |
|------|------|
| `{{related}}` | YAML 列表格式的相关条目双链 |

推荐写法：

```markdown
{{#if related}}相关:
{{related}}
{{/if}}
```

## 10. 角色变量

插件最多提供 9 个角色槽位。

可用变量模式：

- `{{character1}}` 到 `{{character9}}`
- `{{characterCV1}}` 到 `{{characterCV9}}`
- `{{characterPhoto1}}` 到 `{{characterPhoto9}}`

如果角色数量不足，对应变量会为空。

## 11. 已保留但默认不建议使用的兼容变量

| 变量 | 说明 |
|------|------|
| `{{note_link}}` | 当前固定为空，仅为兼容旧模板保留 |

新模板不要再依赖它。

## 12. 推荐模板结构

推荐把模板分成三层：

1. frontmatter：结构化字段，便于 Dataview / Properties 使用
2. 信息卡片：展示 Bangumi 数据与评分
3. 正文区：保留 `记录` 和 `感想`

示例：

```markdown
---
中文名: "{{name_cn}}"
原名: "{{name}}"
阅读状态: {{my_status}}
tags:
{{tags}}
评分: {{my_rate}}
版本:
渠道: "实体"
已购: false
Bangumi评分: "{{rating}}"
Bangumi链接: "{{bangumi_url}}"
---

# {{name_cn}}

> [!abstract]+ 简介
> {{summary}}

{{#if volumes_display}}
## 卷列表

{{volumes_display}}
{{/if}}

## 记录

## 感想
```

## 13. 设计建议

### 推荐

- 对可能为空的块使用 `{{#if ...}}`
- 标签优先使用 YAML 列表写法
- 本地自定义属性直接写在 frontmatter
- 需要默认值的字段直接在模板中写字面量
- 给正文里的用户内容保留稳定结构。内置模板默认仍然使用 `记录` / `感想` 标题，便于阅读和迁移

### 不推荐

- 依赖旧文档里未实现或已移除的变量名
- 把本地自定义默认值继续寄希望于插件设置面板
- 在 frontmatter 中混用太多互斥命名
- 把长文本用户内容塞进动态属性弹窗字段

## 14. 常见问题

### 为什么某个 frontmatter 字段没有出现在动态属性弹窗里？

通常是以下原因之一：

- 该字段用了模板变量，例如 `作者: "{{author}}"`
- 该字段属于插件自动写入字段
- 该字段虽然用了模板变量，但这些变量都能由 Bangumi / 同步流程自动提供
- 该字段不在 frontmatter 中

### 为什么点“跳过”后某个字段还是空的？

因为模板里本来就是空值，例如：

```markdown
存储:
```

“跳过”不会自动猜值，只会保留模板默认值。

### 如何让一个字段显示为开关而不是文本框？

把默认值写成：

```markdown
已购: false
```

或：

```markdown
已购: true
```

### 如何让字段默认显示某个固定文本？

直接写成字面量：

```markdown
平台: "Steam"
渠道: "实体"
```
