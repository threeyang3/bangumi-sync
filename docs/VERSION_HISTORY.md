# 版本历史

## v6.9.1

### 修复
- 修复“同步状态范围选择”弹窗布局异常，恢复完整的标题、双栏字段区域和底部操作按钮
- 调整范围选择弹窗样式实现，避免在 Obsidian modal 容器内出现内容裁切、按钮缺失和滚动区域异常

## v6.9.0

### 新增
- 同步状态恢复为统一入口，执行前先弹出字段范围选择窗口，可分别勾选用户数据和平台数据字段

### 改进
- 控制面板功能栏重构为默认态/选中态两套固定 8 键布局，移动端继续 4x2，桌面端继续单行
- 同步状态差异弹窗支持按本轮勾选字段增量展示与执行，不再绑定用户/平台二选一模式

### 清理
- 移除旧的短评同步、标签同步与平台专用状态同步入口残留
- 删除只为旧拆分方案服务的翻译键、字段预设与临时清理脚本

## v6.8.11

### 改进
- 将“同步状态”拆分为“同步用户数据”和“同步平台数据”两个独立入口，控制面板按钮和命令入口分别对应两条链路
- 平台同步不再基于本地完结状态跳过候选，改为统一对支持的平台字段执行对比

### 变更
- 默认模板、模板变量文档和本地 frontmatter 平台写回中移除 `连载状态` 字段
- 平台数据同步只保留平台字段更新，不再依赖本地 `连载状态` 属性做状态修正

## v6.8.10

### 修复
- 动画和三次元平台状态解析兼容 `播放开始 / 播放结束`，修复已完结条目在状态同步中被误判为 `连载中` 的问题
- 平台数据差异默认选择云端，执行状态同步时会默认回写云端平台字段修正

### 改进
- 抽出平台字段 diff helper，减少状态同步 service 中重复对象拼装
- 新增 infobox 媒体起止日期解析回归测试和平台字段默认决策测试，纯逻辑测试扩展到 30 项

## v6.8.9

### 改进
- 抽出 `src/document/` 文档服务层，统一本地条目的短评、frontmatter、section、平台字段与单集状态快照读写
- 抽出 `StatusSyncService`，把状态同步的 diff 构建、后台补全、默认决策和执行写回从 UI 编排层中收口
- 新增 frontmatter 访问与 markdown section 读写测试，纯逻辑测试扩展到 26 项

### 修复
- `statusSyncModal` 不再回退到全局 `window`，继续符合 Obsidian 的 popout window 兼容规范

## v6.8.8

### 改进
- 状态同步改为“快速首屏 + 后台补全”，先展示用户数据差异，再后台增量加载单集状态与平台数据

- 本地 -> 云端的评分、短评、标签、收藏状态改为尽量合并成一次收藏更新请求，减少状态同步时的网络往返

- 短评 callout 的读取、导入对比和模板渲染继续统一为同一套正文来源，避免多段短评表现不一致
