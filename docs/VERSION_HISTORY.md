# 版本历史

## v5.8.3

### 改进
- 全局 UI 主题大统一：将“添加收藏”弹窗、标签、状态角标等所有遗留组件统一为纸张感暖色主题及 12px/999px 圆角。
- 搜索界面画龙点睛：移动端“添加”按钮改为自适应宽度并右对齐，增强了 Sticky 顶栏的遮挡质感与清晰度。
- 优化移动端卡片间距与对齐逻辑，视觉体验更加精致。

## v5.8.2

### 改进
- 深度优化搜索界面审美：统一搜索框、选择器与按钮的圆角及色调，强制继承“纸张感”暖色主题，消除视觉补丁感。
- 精修搜索结果卡片：优化标题层级显示（中文加粗，原名缩小），使列表视觉更加轻盈、信息重心更明确。
- 对齐控件规格：修复移动端搜索栏控件高度不一致的问题，增强触控友好度。

## v5.8.1

### 改进
- 深度优化桌面端 UI 审美：通过统一背景色调与边框风格，消除了面板“分层感”和“套娃”视觉。
- 修复桌面端双滚动条问题：锁定弹窗外层溢出，仅允许内容区域（列表/表格）滚动。
- 优化弹窗呼吸感：调整桌面端内边距，使布局更加美观、一体化。

## v5.7.0

### 新增功能
- **API 重试机制**：请求遇到 429 或 5xx 错误时自动重试 3 次（指数退避 1s→2s→4s）
- **错误详情展示**：同步完成后可折叠查看每个失败条目的具体错误信息
- **同步并发数可配置**：设置页新增并发数滑块（1-5），控制同时处理的条目数量
- **新增 11 个模板变量**：`from`（原作）、`musicMake`（音乐制作）、`audioDirector`（音响监督）、`artDirector`（美术监督）、`animeChief`（总作画监督）、`start`（开始日期）、`end`（结束日期）、`script`（剧本）、`art`（原画）、`producer`（制作人）、`price`（售价）

### 改进
- **并发同步统一**：控制面板同步和手动同步均已支持并发处理
- **双向链接批量更新**：相关条目链接按目标文件分组，每个文件只读写一次
- **路径查找索引**：扫描本地文件夹后构建 id→path 反转索引，路径解析从 O(N) 降至 O(1)
- **分页动态间隔**：收藏列表获取前 5 页间隔 50ms，之后 100ms，支持取消信号
- **路径模板渲染统一**：8 次独立替换合并为单次正则匹配
- **清理幽灵变量**：移除 `voice1`~`voice9`（已被 `characterCV` 替代）

## v5.6.0

### 新增功能
- **性能优化**：文件扫描优先使用 metadataCache，减少不必要的文件读取
- **路径解析优化**：相关条目路径解析避免遍历所有文件
- **模板渲染优化**：使用单次正则匹配，减少匹配次数
- **并发同步**：批量同步支持并发处理，默认并发数为 3

### 改进
- **类型标签修复**：书籍类型现在显示具体分类（小说、漫画、画集），而非统一显示"书籍"
- **其他类型优化**：音乐、游戏、三次元类型也优先使用细分类别
- **代码质量**：符合 Obsidian 插件规范，通过 lint 和 build 检查

## v5.3.4

### 改进
- **移动端搜索弹窗优化**：搜索框与搜索/清除按钮改为同一行紧凑布局，筛选类型与排序选择保持同一行
- **移动端搜索结果优化**：结果卡片保留左侧封面缩略图，标题独占内容行，元数据、收藏状态和添加/编辑按钮同排显示
- **移动端分页修复**：控制面板分页改为直接三列布局，避免移动端 WebView 对 `display: contents` 支持异常导致按钮消失
- **移动端状态文案修复**：修正缓存数据加载状态中重复显示“共”的问题
- **移动端控制面板优化**：条目列表改为卡片布局，选择框在左列，标题、元数据、短评、标签在右侧内容区分行显示
- **移动端操作栏压缩**：顶部操作按钮改为更紧凑的四列布局，并补充边框按钮样式
- **移动端状态栏优化**：`已选` 计数移动到加载状态文字右侧，不再单独占用一整行
- **移动端分页优化**：底部分页压缩为“上一页 / 页码 / 下一页”单行布局，并为分页按钮补充边框
- **移动端元数据紧凑排列**：类型、状态、评分、同步状态横向排列，用分隔符连接
- **移动端标题优化**：隐藏原标题（日语标题），只显示中文名
- **移动端列宽修复**：表格宽度限制为容器宽度，标题、短评、标签均在行末截断，无需左右滑动
- **移动端选择框优化**：使用自定义勾选样式，避免 iOS 默认勾选符号过小
- **移动端条目操作优化**：已同步条目的“打开”和“笔记”按钮移动到选择框左列纵向排列
- **移动端关闭按钮优化**：控制面板关闭按钮避开 iOS 顶部状态栏安全区
- **滑动关闭手势优化**：只在表格滚动到顶部时才允许下拉关闭，避免滚动时误触发关闭面板

## v5.2.0

### 新增功能
- **统一自定义属性模块**：插件改为先读取模板 frontmatter 的全部属性，再过滤掉可由 Bangumi 自动提供的字段；剩余字段统一视为本地自定义属性
- **动态填写窗口扩展**：`同步收藏`、控制面板同步、`搜索并添加` 现在都会按模板动态弹出自定义属性填写窗口
- **列表型自定义属性**：支持 `资源属性: []` 这类列表字段，输入时可用英文逗号拆分为 YAML 数组

### 改进
- **模板默认值来源统一**：本地自定义属性默认值完全来自模板，不再依赖设置面板硬编码
- **评分明细地位统一**：内置作者模板中的评分明细改为普通 frontmatter 自定义属性写法，不再默认使用 `{{rating_xxx}}`
- **快速同步 / 自动同步行为明确**：不会弹出填写窗口，自定义属性只会使用模板默认值或保持为空
- **用户数据结构收敛**：导出 / 导入 / 强制同步继承统一按辨识属性、自定义属性、记录/感想三部分处理

### 修复
- 修复收藏状态查询错误导致的 `/collections/...` 404 刷屏与已收藏状态误判
- 修复 `资源属性` 等字段被模板条件块误判而不出现在填写窗口的问题
- 修复数组型自定义属性在导入、覆盖、继承时被错误退化为字符串的问题
